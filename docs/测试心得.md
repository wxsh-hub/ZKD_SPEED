# 接口测试心得

## 环境信息

- **后端端口**: 9090
- **Context Path**: `/api/ragent`
- **数据库**: `222` (MySQL)
- **认证方式**: SaToken，token 通过 `Authorization` 请求头传递

## 登录获取 Token

```bash
curl -s -X POST http://localhost:9090/api/ragent/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

返回示例：
```json
{
  "code": "0",
  "data": {
    "userId": "2001523723396308993",
    "role": "admin",
    "token": "e1a3296c7ca24bf091d72a95cc3dc944",
    "avatar": "..."
  }
}
```

取 `data.token` 字段作为后续请求的凭证。

## 请求头格式

```
Authorization: <token>
```

**注意**: 不是 `token`，是 `Authorization`。这个配置在 `application.yaml` 中：
```yaml
sa-token:
  token-name: Authorization
```

## URL 拼接规则

所有接口路径 = `http://localhost:9090/api/ragent` + 接口路径

例如创建项目：`POST http://localhost:9090/api/ragent/script/project`

## 常见错误码

| code | 含义 | 排查方向 |
|------|------|----------|
| `A000001` | 未登录或token过期 | 重新登录获取token，检查Authorization头 |
| `B000001` | 系统执行出错 | 通常是数据库表不存在或SQL错误，检查后端日志 |
| `A000002` | 权限不足 | 当前用户非admin角色 |

## 新模块上线必做

1. **执行建表 SQL** — 新模块的数据库表需要手动执行 `resources/database/schema_table.sql` 中对应的建表语句
2. **确认 MapperScan** — `RagentApplication.java` 的 `@MapperScan` 中包含新模块的 mapper 包路径
3. **重启后端** — 代码变更后需要重启才能生效

## 测试模板

```bash
# 1. 登录
TOKEN=$(curl -s -X POST http://localhost:9090/api/ragent/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 2. 调用接口
curl -s http://localhost:9090/api/ragent/<你的接口路径> \
  -H "Authorization: $TOKEN"
```

## 查看日志

后端日志位于：
- `bootstrap/logs/ragent-info.log` — 一般日志
- `bootstrap/logs/ragent-error.log` — 错误日志
- `bootstrap/logs/ragent-warn.log` — 警告日志

遇到 `B000001` 错误时，先查 error log 定位具体异常。
