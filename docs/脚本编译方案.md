# 脚本编译方案

## 概述

用户在前端标注截图、配置操作步骤后，系统将标注数据转换为 Python 脚本，再通过 Nuitka 编译为独立的 Windows EXE 文件。

## 编译流程

### 1. 代码生成

`ScriptCodeGenerator.java` 将数据库中的步骤数据（`ScriptStepDO`）转换为 Python 脚本代码。

生成的脚本基于 **ctypes** 调用 Windows API，不依赖任何第三方库：

- 鼠标操作：`user32.SetCursorPos`、`user32.mouse_event`
- 键盘输入：`user32.SendInput` + `KEYEVENTF_UNICODE`（支持中文）
- 滚轮：`MOUSEEVENTF_WHEEL` / `MOUSEEVENTF_HWHEEL`

相比 pyautogui 方案，ctypes 方案：
- 无第三方依赖，编译产物体积小（~4MB vs ~65MB）
- 不会被杀毒软件隔离（纯系统 API 调用）

### 2. Nuitka 编译

`ScriptServiceImpl.doBuild()` 异步执行编译：

```
python -m nuitka --standalone --onefile --disable-console --assume-yes-for-downloads script.py
```

- `--standalone`：打包所有依赖
- `--onefile`：输出单个 EXE 文件
- `--disable-console`：无控制台窗口
- Python 路径通过 `script.python-path` 配置（需纯 ASCII 路径，Nuitka 不支持中文路径）

编译产物为独立 EXE，**不需要用户安装 Python**。

### 3. 上传存储

编译完成后，EXE 文件上传到腾讯云 COS：

- Bucket：`wxsh-1361118632`
- Region：`ap-guangzhou`
- 路径格式：`script-builds/{projectId}/{taskId}.exe`
- 访问域名：`https://wxsh-1361118632.cos.ap-guangzhou.myqcloud.com`

## 关于 Windows SmartScreen 警告

Nuitka 编译的 EXE 没有数字签名，首次下载运行时 Windows SmartScreen 会弹出"Windows 已保护你的电脑"警告。用户点击"仍要运行"即可，这是所有未签名 EXE 的正常行为。

杀毒软件的行为检测不会拦截，因为脚本只调用标准 Windows API（模拟鼠标/键盘），与按键精灵、AutoHotkey 等自动化工具行为一致。
